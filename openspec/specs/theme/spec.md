# Theme Specification

> Generated by openlore v1.0.0 on 2026-07-11 03:51
> Reviewed & corrected by hand on 2026-07-11

## Purpose

Manages the application's theme settings, including light/dark mode toggling and persistence.

## Requirements

> `web/src/useTheme.ts`

### Requirement: ApplyTheme

> Implementation: `useTheme` in `web/src/useTheme.ts` · confidence: reviewed

The system SHALL resolve and apply the effective light/dark theme from, in priority order: the
user's stored preference (localStorage), then the configured default (`branding.defaultTheme`,
which may be "light", "dark", or "system"), falling back to "system". A "system" preference
follows `prefers-color-scheme` and reacts to OS changes. The resolved theme is applied as a
`data-theme` attribute on the target root element (document root, or the widget root when
embedded).

#### Scenario: UserPrefersDarkMode
- **GIVEN** User has a stored dark preference
- **WHEN** the theme is resolved
- **THEN** Dark theme is applied to the root element

#### Scenario: SystemPreferenceFollowed
- **GIVEN** No stored preference and the effective default is "system"
- **WHEN** the theme is resolved
- **THEN** The OS `prefers-color-scheme` value is applied
- **AND** an OS-level change updates the applied theme without a reload

### Requirement: ToggleTheme

> Implementation: `useTheme` in `web/src/useTheme.ts` · confidence: reviewed

The system SHALL toggle between light and dark themes and persist the new preference in
localStorage. The toggle can be disabled by config (`branding.allowThemeToggle: false`), e.g.
when a host application embedding the widget drives the theme itself.

#### Scenario: ToggleFromLightToDark
- **GIVEN** Current theme is light and the toggle is enabled
- **WHEN** toggle is called
- **THEN** Dark theme is applied to the root element
- **AND** The new preference is stored in localStorage

#### Scenario: ToggleFromDarkToLight
- **GIVEN** Current theme is dark and the toggle is enabled
- **WHEN** toggle is called
- **THEN** Light theme is applied to the root element
- **AND** The new preference is stored in localStorage

## Technical Notes

- **Dependencies**: localStorage, `window.matchMedia("(prefers-color-scheme: dark)")`, React state/effects
