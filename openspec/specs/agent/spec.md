# Agent Specification

> Generated by openlore v1.0.0 on 2026-07-11 03:51
> Reviewed & corrected by hand on 2026-07-11

## Purpose

Manages the state and communication with the agent backend, including WebSocket connections, message
handling, and state updates.

## Requirements

> `web/src/useAgent.ts`

### Requirement: InitializeAgent

> Implementation: `useAgent` in `web/src/useAgent.ts` · confidence: reviewed

The system SHALL initialize the agent client by opening a WebSocket to the server and setting up
reducer-based state management. On connect, the server sends a full state snapshot (session id,
items, models, branding, queue) that replaces local state.

#### Scenario: SuccessfulInitialization
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A reachable server URL
- **WHEN** the useAgent hook mounts
- **THEN** A WebSocket connection is established, `connected` becomes true, and the snapshot is applied

#### Scenario: FailedInitialization
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** An unreachable server URL
- **WHEN** the useAgent hook mounts
- **THEN** `connected` stays false and the client retries the connection after a delay

### Requirement: SendMessage

> Implementation: `send` / `prompt` in `web/src/useAgent.ts` · confidence: reviewed

The system SHALL send ClientMessage frames (prompt with optional images, abort, session and tree
operations…) to the server over the WebSocket as JSON.

#### Scenario: SuccessfulMessageSend
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** An active WebSocket connection
- **WHEN** an action (e.g. `prompt`) is called
- **THEN** The corresponding ClientMessage is serialized and sent to the backend

#### Scenario: FailedMessageSend
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** No active WebSocket connection
- **WHEN** an action is called
- **THEN** The message is not sent and the UI reflects the disconnected state

### Requirement: HandleNotification

> Implementation: `reducer` in `web/src/useAgent.ts` · confidence: reviewed

The system SHALL handle ServerMessage frames received from the backend (streaming deltas, tool
events, session_replaced snapshots, tree updates, extension UI requests, errors) by dispatching
them to the reducer.

#### Scenario: SuccessfulNotificationHandling
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A valid ServerMessage frame
- **WHEN** it is received on the WebSocket
- **THEN** The message is dispatched and state is updated accordingly

#### Scenario: FailedNotificationHandling
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A frame that is not valid JSON
- **WHEN** it is received on the WebSocket
- **THEN** The frame is ignored without crashing the client

### Requirement: UpdateState

The system SHALL update client state exclusively through the reducer, keyed by ServerMessage type.
Snapshot-bearing messages (`session_replaced`) replace items wholesale and invalidate derived
state such as the cached conversation tree.

#### Scenario: SuccessfulStateUpdate
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A known message type and a valid payload
- **WHEN** the reducer processes it
- **THEN** The state is updated accordingly

#### Scenario: SnapshotInvalidatesDerivedState
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A cached conversation tree in state
- **WHEN** a `session_replaced` snapshot arrives
- **THEN** The tree is reset to null so consumers refetch it

## Technical Notes

- **Dependencies**: WebSocket, React (useReducer/useEffect/useRef/useCallback), `@pi-outpost/shared` wire protocol
