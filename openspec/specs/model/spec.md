## Requirements

### Requirement: ClientMessageValidation

The system SHALL validate ClientMessage according to these rules:
- type must be one of: prompt, abort, set_model, set_thinking, new_session, switch_session, delete_session, list_sessions, compact, list_directory, read_file, search_files, list_tree, navigate_tree, fork_session
- When type is 'prompt', text is required
- When type is 'set_model', provider and id are required
- When type is 'set_thinking', level is required
- When type is 'switch_session' or 'delete_session', path is required
- When type is 'list_directory' or 'read_file', requestId is required
- When type is 'search_files', query and requestId are required
- When type is 'navigate_tree' or 'fork_session', entryId is required

#### Scenario: SendPromptMessage
- **GIVEN** Client has text to send
- **WHEN** Client sends message with type 'prompt'
- **THEN** Server receives prompt message with text content

#### Scenario: AbortOperation
- **GIVEN** Client has ongoing operation
- **WHEN** Client sends message with type 'abort'
- **THEN** Server aborts the ongoing operation

#### Scenario: SetModel
- **GIVEN** Client knows provider and model id
- **WHEN** Client sends message with type 'set_model'
- **THEN** Server sets the model for subsequent operations

#### Scenario: SetThinkingLevel
- **GIVEN** Client knows desired thinking level
- **WHEN** Client sends message with type 'set_thinking'
- **THEN** Server sets the thinking level for processing

### Requirement: AppConfigValidation

The system SHALL validate AppConfig according to these rules:
- baseCwd must be a valid path
- baseCwd must be an absolute path
- branding must be a valid BrandingConfig object
- sandbox must be a valid SandboxConfig object

#### Scenario: LoadApplicationConfiguration
- **GIVEN** A valid baseCwd path
- **WHEN** loadConfig(baseCwd) is called
- **THEN** AppConfig is loaded with branding, sandbox, and other settings
- **AND** If baseCwd is invalid, throw an error
- **AND** If branding or sandbox are invalid, throw an error

### Requirement: BrandingConfigValidation

The system SHALL validate BrandingConfig according to these rules:
- theme must be one of the predefined themes
- logo must be a valid URL if provided
- favicon must be a valid URL if provided
- primaryColor must be a valid color if provided
- secondaryColor must be a valid color if provided

#### Scenario: ValidBrandingConfigAccepted
- **GIVEN** A valid BrandingConfig object with all required fields
- **WHEN** The object is validated
- **THEN** Validation passes with no errors

### Requirement: SandboxConfigValidation

The system SHALL validate SandboxConfig according to these rules:
- enabled must be a boolean
- features must be an array of strings if provided
- models must be an array of ModelConfig objects if provided

#### Scenario: ValidSandboxConfigAccepted

- **WHEN** The object is validated
- **THEN** Validation passes with no errors
